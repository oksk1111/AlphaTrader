1단계: SSH 접속 (로컬 터미널)
키 파일이 있는 폴더에서 아래 명령어로 서버에 접속합니다.

# 키 파일 권한 확인 (이미 했다면 생략 가능)
chmod 400 ssh-key-2026-01-07.key

# 서버 접속 (Oracle Linux인 경우 ubuntu 대신 opc 사용)
ssh -i ssh-key-2026-01-07.key ubuntu@158.180.81.25


2단계: 최신 코드 업데이트 (서버 접속 후)
서버에 접속했다면, 프로젝트 폴더로 이동하여 방금 푸시한 코드를 내려받습니다.

# 프로젝트 폴더로 이동
cd AlphaTrader

# 최신 코드 당겨오기 (커밋된 한국 주식 기능 등)
git pull

# (선택) 혹시 모를 의존성 업데이트
source venv/bin/activate
pip install -r requirements.txt


3단계: 봇 재시작
코드가 바뀌었으므로 봇 서비스를 재시작해야 새로운 로직(한국/미국 듀얼 모드)이 적용됩니다.

# 1. 관리자 권한으로 설정 파일 다시 읽기
sudo systemctl daemon-reload

# 2. 봇 서비스 재시작
sudo systemctl restart alphatrader-bot
sudo systemctl restart alphatrader-dashboard

# 3. 상태 확인
sudo systemctl status alphatrader-bot
sudo systemctl status alphatrader-dashboard

(로그에서 Quit 혹은 q를 눌러 빠져나올 수 있습니다)

4단계: 클라우드 터널링(웹 대시보드) 시작
웹 대시보드(dashboard.py)를 백그라운드로 띄우고, 클라우드플레어 터널을 연결합니다.

# 1. 기존에 실행 중인 대시보드/터널 정리 (깔끔하게)
pkill -f "streamlit"
pkill -f "cloudflared"

# 2. 대시보드 실행 (백그라운드)
nohup venv/bin/streamlit run dashboard.py > dashboard.log 2>&1 &

# 3. 클라우드플레어 터널 실행 (백그라운드)
# (이미 cloudflared가 설치되어 있다고 가정)
nohup ./cloudflared tunnel --url http://localhost:8501 > cloudflared.log 2>&1 &

# 4. 터널 주소 확인 (약 5초 후 실행)
sleep 5 && cat cloudflared.log | grep "trycloudflare.com"


# oracle cloud 접속
ssh -i ssh-key-2026-01-07.key -f -N -L 8501:localhost:8501 ubuntu@158.180.81.25


# 로컬서버 활성화
1. 첫 번째 터미널: 거래 봇 실행
봇은 백그라운드에서 계속 돌면서 매매를 수행합니다.
# 가상환경 활성화 (이미 되어 있다면 생략 가능)
source venv/bin/activate
# 봇 실행
python run_bot.py

2. 두 번째 터미널: 웹 대시보드 실행
현재 상태를 확인하기 위한 웹 화면을 띄웁니다.
# 가상환경 활성화
source venv/bin/activate
# 대시보드 실행
streamlit run dashboard.py

명령어를 실행하면 브라우저가 자동으로 열리며 대시보드(http://localhost:8501)가 뜹니다.
