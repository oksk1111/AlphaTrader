# 📁 문서 1. [기획서] Project Alpha Trader & Stock Hunter

## 1. 프로젝트 개요

- **프로젝트 명:** Alpha Trader (미국/한국 주식 자동매매 봇)
- **목표:** 
    1. **미국 시장**: 빅테크 및 레버리지 ETF를 중심으로 추세 추종. **3배 ETF 제한 없음.**
    2. **한국 시장**: ETF + 개별 종목 발굴(가치주/수급주) 전략. **1000만원 미만 시 레버리지 ETF 제한.**
    3. **AI 기반**: 다중 LLM 합의 시스템으로 뉴스 및 수급 분석을 통해 유망 종목을 실시간으로 포착.
- **핵심 대상:**
    - **US**: TQQQ, SOXL, NVDA, MSFT, AAPL 등 빅테크 및 ETF (3배 레버리지 항상 가능)
    - **KR**: KODEX 레버리지, 개별 우량주 및 거래량 급등주 (KRW 1000만원 이상 시 레버리지)
    - **자동 포트폴리오 (Dynamic Portfolio)**: 미리 정의된 30여 개의 고품질 ETF 풀(K-방산, 가치업, 기술주 등) 중 모멘텀/안정성 지표가 우수한 상위 N개(예: 10~15개) ETF를 실시간 필터링하여 자동으로 교체/삭제 편입. (자동 리밸런싱 지원)
- **운영 시간:**
    - 한국장: 09:00 ~ 15:30
    - 미국장: 23:30 ~ 06:00
- **수익 모델:** 사용자 투자 무료, 광고 수익 기반

## 2. 투자 전략 (The Strategy)

### 2.0. 참고 문서 반영 개선안 (daily_stock_analysis 스타일)

레퍼런스 문서의 운영 방식(설정 중심, 요약 중심, 주기 점검)을 반영해 다음 원칙으로 개선합니다.

1. 설정 중심 정책화
- AI 합의 규칙을 코드 하드코딩이 아닌 설정(`user_config.json`)으로 제어
- 운영 환경마다 보수/공격 성향을 정책값으로 즉시 조정 가능

2. 의사결정 요약 표준화
- 대시보드 개요에 "오늘의 의사결정 요약(1문장)" 고정 노출
- 운영자가 장 상태/최근 주문/신호 강도를 즉시 파악하도록 단일 결론 제공

3. 주기 검증 루프 강화
- 주간 백테스트 자동 생성(일요일 07:00 KST)
- 실거래 히스토리 기반 성과/리스크 지표를 정기 점검

### 2.0.1. AI 합의 정책 설정 스키마

`user_config.json` 예시:

```json
{
    "llm_consensus": {
        "crash_veto": true,
        "min_successful_llms": 2,
        "required_buy_ratio": 0.6,
        "unknown_fallback_hold": true,
        "tie_breaker": "persona"
    }
}
```

| 키 | 기본값 | 의미 |
|----|--------|------|
| `crash_veto` | `true` | LLM 중 1개라도 CRASH면 매수 차단 |
| `min_successful_llms` | `1` | 합의 유효 판정 최소 응답 수(쿼럼) |
| `required_buy_ratio` | `0.5` | 매수 승인에 필요한 찬성 비율 |
| `unknown_fallback_hold` | `true` | 전체 실패 시 보수적 관망 여부 |
| `tie_breaker` | `persona` | 임계 동률 시 처리(`persona`/`buy`/`hold`) |

### 2.1. 시장별 차별화 전략

#### 🇺🇸 US 시장 전략
- **3배 레버리지 ETF 항상 사용 가능** (TQQQ, SOXL, TECL 등)
- 미국은 레버리지 ETF 투자에 예탁금 제한이 없음
- 대형주 + 레버리지 ETF 혼합 포트폴리오
- VBO(변동성 돌파) + DCA(분할매수) 하이브리드

#### 🇰🇷 KR 시장 전략
- **예탁금 1,000만원 미만**: 1배수 ETF + 개별 우량주만 거래
- **예탁금 1,000만원 이상**: 2배 레버리지 ETF 추가 가능
- 스캐너 기반 동적 종목 발굴
- **개선된 리스크 관리**: 개별 주식의 높은 변동성 감안

### 전략 모드 선택

| 모드 | 설명 | 적합한 시장/대상 |
|------|------|------------|
| **VBO (변동성 돌파)** | 목표가 돌파 시 매수, 장 마감 매도 | 지수 ETF, 대형주 |
| **Scanner (종목 발굴)** | 거래량 급등, 수급 상위 종목 실시간 포착 | 한국 개별주, 급등주 |
| **DCA (분할매수)** | 매일 정해진 금액 분할 매수 | 장기 투자 (빅테크) |

### Step 1. [Trend Filter & Discovery] 종목 선정 및 추세 판단

**"수급이 몰리고 추세가 살아있는 종목을 산다."**

- **Target List (기본):** 사전에 정의된 우량주 및 ETF
- **Scanner (동적):** 장중 거래량 급등(Volume Spike) 및 거래대금 상위(Blue Chip Surge) 종목을 실시간 포착
- **AI Check:** 다중 LLM 합의를 통해 시장 붕괴 조짐이 있는지 확인

### Step 2. [Entry Trigger] 진입 타점

#### VBO 모드 (변동성 돌파)
- **공식:** 매수 주문가 = $Today Open + (Yesterday Range \times K)$
    - $Yesterday Range = (Yesterday High - Yesterday Low)$
    - $K = 0.5$ (노이즈 필터링 상수)

#### DCA 모드 (분할매수)
- 매일 고정 금액 투자 (가용 자금의 5%)
- 장 시작 30분 후 자동 매수 (시가 안정화 후)

#### HYBRID 모드 (하이브리드)
- **추세 상승 (20MA 위):** VBO 전략 실행
- **추세 횡보/하락 (20MA 아래):** DCA 전략 실행 (소량)

### Step 3. [AI Multi-LLM Consensus Filter] 재료 검증 (진입 직전)

**"기술적 신호가 떴지만, 혹시 오늘 파월 의장이 금리 인상 발언을 했는가?"**

- **다중 LLM 합의 시스템:**
    - **Gemini** (Google): 빠른 뉴스 해석, 영어 역량 탁월
    - **Grok** (xAI): X(트위터) 실시간 감성 분석
    - **DeepSeek** (DeepSeek): 중국/아시아 시장 관점 분석
    - **Groq** (Groq): 초고속 추론 (Llama/Mixtral 호스팅), 보조 의견
- **합의 방식:**
    - 각 LLM이 독립적으로 `risk_level` (HIGH/LOW)과 `can_buy` (true/false) 판단
    - **다수결(Majority Vote):** 과반수 이상이 매수 추천 시 매수
    - 한 LLM이라도 "CRASH" 판정 시 매수 차단

### Step 4. [Constraints & Fallback] 제약 사항 및 대응

**🇺🇸 US 시장: 3배 레버리지 ETF 제한 없음**
- 미국 시장에서는 3배 레버리지 ETF 투자에 별도의 예탁금 제한이 없음
- 따라서 US 시장은 항상 전체 ETF 목록(3x 포함)으로 거래

**🇰🇷 KR 시장: 예탁금 1,000만원 기준**
- **예탁금 1,000만원 미만:** 1배수 ETF + 개별 우량주만 거래
- **예탁금 1,000만원 이상:** 2배 레버리지 ETF 추가 가능
- 자동으로 예탁금을 확인하여 모드 전환

### Step 5. [Exit Rule] 청산 전략

#### 시장별 차별화된 리스크 관리 (v2.3 갱신)

| 항목 | US 시장 | KR 시장 (개별주) | KR 시장 (ETF) |
|------|---------|------------------|---------------|
| **Stop Loss** | -5.0% | **-7.0%** | -5.0% |
| **Trailing Stop 활성** | +5.0% | **+7.0%** | +5.0% |
| **Trailing Stop 하락** | -3.0% | **-4.0%** | -3.0% |
| **Gap Down 차단** | -5.0% | **-6.0%** | -5.0% |
| **Portfolio Drawdown 차단** | -7.0% | -7.0% | -7.0% |

#### 1. Stop Loss (손절매)
- **US/ETF:** -5.0% 시 매도 발동 → `safe_sell()` (시장가 3회 → 지정가 fallback 2회)
- **KR 개별주:** -7.0% 시 매도 발동 (단기 변동성 흡수)
- 매도 실패 시 `STOP_LOSS_MAX_RETRY_CYCLES=5` 사이클 한도 도달 시 강제 정리 (스팸 방지)

#### 2. Trailing Stop (트레일링 스탑)
- **US/ETF:** +5.0% 초과 후 고점 대비 -3.0% 하락 시 매도
- **KR 개별주:** +7.0% 초과 후 고점 대비 -4.0% 하락 시 매도

#### 3. Time Cut (종가 청산)
- 장 마감 10분 전 보유 종목 시장가 매도
- DCA/Swing 전략은 종가 청산 예외 (장기 보유)

#### 4. Gap Down Defense (갭다운 방어)
- 전일 종가 대비 급락 시 매수 차단 — 단 **변동성 반등 트리거** 충족 시 50% 수량으로 역방향 매수

#### 5. Portfolio Circuit Breaker
- 전체 포트폴리오 평가손실 -7.0% 이상 시 신규 매수 중단

#### 6. ⚡ Oversold Rebound Buy (변동성 반등 매수, v2.3 신규)
- 누적 하락/갭다운 ≥5% + 당일 저점 대비 반등 ≥1% + 현재가 ≥ 5MA 충족 시
- 매수 차단 가드를 무효화하고 **50% 수량으로 역방향 매수** (세션당 1회)
- AI veto 우회 (오버솔드 구간 진입 기회 확보)

## 4. 시스템 아키텍처

```
BrokerInterface (ABC)
├── get_current_price(ticker, exchange)
├── get_daily_ohlc(ticker, exchange)

├── KISBroker (한국투자증권)

├── NHBroker (NH투자증권) [예정]
└── AlpacaBroker (Alpaca) [예정]
```

### 다중 LLM 합의 시스템

```
MultiLLMAnalyst
├── GeminiAnalyst (Google)
├── GrokAnalyst (xAI)
├── DeepSeekAnalyst (DeepSeek)
└── GroqAnalyst (Groq/Llama)

합의 로직:
1. 각 LLM에 동일한 뉴스 + 프롬프트 전달
2. 각 LLM이 독립적으로 risk_level + can_buy 판단
3. 다수결 투표 (Majority Vote)
4. CRASH 판정이 하나라도 있으면 매수 차단

### 운영 결론(Decision Brief) 생성 레이어

```
Dashboard Payload Builder
├── 상태(status.marketStatus)
├── 최근 주문(activity.lastOrder)
├── 최상위 신호(signals[0])
└── 1문장 결론(status.decisionBrief) 생성
```

결론 예시:
- "관망 권장: 시장 마감 상태입니다."
- "주의: 최근 주문 실패가 감지되었습니다."
- "공격 관찰: SOXL 신호 강도가 높습니다."
```

- 레버리지 ETF는 변동성이 높아 일반 종목 기준 필터에 과도하게 걸림

**해결책:**

## v2.3 (2026.05.23) — 수익 극대화 & 매도 안정화

### 🚨 CRITICAL — 손절매 매도 주문 실패 무한 알람 버그 수정

**문제점:**
- 005930, 000660 등 KR 개별주에서 손절매 발동 후 매도 주문이 반복적으로 실패하며 텔레그램 알람이 무한히 누적됨
- 원인:
  1. **Rule No.0 (재시도 루프)** 가 `data['buys']` (메모리 캐시) 만 사용 → 실제 KIS 잔고와 mismatch 시 영원히 실패
  2. **장 외 시간 매도 시도** — KR 정규장(09:00–15:30) 종료 후에도 시장가 주문 시도 → KIS 거부
  3. **시장가 단일 의존** — `ORD_DVSN=01` 거부 시 fallback 없음
  4. **에러 코드 비공개** — `msg_cd`/`rt_cd` 로깅 부재로 원인 추적 불가

**해결책 — `safe_sell()` 통합 매도 헬퍼 도입 (`run_bot.py`):**
1. **잔고 재조회** — 매 시도마다 `kis.get_holding_qty(ticker)` 로 실제 보유 수량 재확인. 0이면 `already_flat` 처리(외부 청산 완료 인식). 캐시 수량 > 실제 보유 시 실제 수량으로 클램프.
2. **시장 시간 가드** — `is_market_open_for(market)` 가드. 장 마감이면 `deferred` 반환하고 알람 보내지 않음.
3. **시장가(3회) → 지정가 fallback(2회)** — `sell_limit_order` (KR) 추가. 호가단위 자동 라운딩 적용 (`_tick_size`). 지정가는 현재가의 -0.5% 할인하여 즉시 체결 유도.
4. **재시도 한도** — `STOP_LOSS_MAX_RETRY_CYCLES=5` 도달 시 강제 `sold_sl` 처리 + 단 1회 운영자 알림 (스팸 방지).
5. **상세 로깅** — `rt_cd`, `msg_cd`, `msg1` 모두 로깅하여 KIS 거부 사유 즉시 식별 가능.

같은 헬퍼가 Stop Loss, Trailing Stop, DCA Stop Loss, Rule No.0 재시도 4곳 모두에서 일관되게 사용됨.

### ⚡ 변동성 반등 매수 (Oversold Rebound Trigger)

**문제:** 큰 하락(갭다운 또는 연속하락 ≥5%) 후 당일 반등이 시작되어도 기존 가드가 무조건 매수를 차단하여 회복 구간 진입 기회를 놓침.

**해결:** `attempt_dca_buy()` 에서 차단 조건이 발동되더라도 아래 모든 조건을 만족하면 차단을 무효화하고 **50% 수량으로 역방향 매수**:
- 누적 하락 또는 갭다운 ≥ `REBOUND_DROP_THRESHOLD_PCT` (기본 5%)
- 당일 저점 대비 반등 ≥ `REBOUND_INTRADAY_BOUNCE_PCT` (기본 1%)
- 현재가가 5MA 이상 (단기 반전 신호)
- 세션 반등 매수 횟수 < `REBOUND_MAX_BUYS_PER_SESSION` (기본 1)
- 반등 매수 트리거 시 AI veto 우회 (오버솔드 진입 기회 확보)
